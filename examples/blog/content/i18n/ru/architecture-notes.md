# Архитектурные заметки

Блоговый пример показывает, как сочетать три переиспользуемые части фреймворка:

1. `pkg/content-markdown` — библиотека markdown, которая предварительно рендерит
   страницы на старте.
2. `pkg/web/locale` — middleware определения локали в контексте запроса.
3. `pkg/cache` — TTL-кеш со стратификацией для мемоизации уже отрендеренных HTML-страниц.

Каждый blog feature включает:

- **Маршруты**: `/`, `/posts/{slug}`.
- **Templ-вьюхи**: страницу списка и страницу одной статьи.
- **Реестр контента**: срез `{slug, title}`, связанный с встроенными `*.md` файлами.

Добавить новый пост — это четыре строки правок: положить markdown-файл,
добавить его запись в реестр в `internal/site/blog/feature.go`, прогнать `templ`
и перезапустить сервер.
