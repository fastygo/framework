# UI8Kit Templ — план развития

Цель: полная библиотека UI-компонентов на Go Templ с **100% соответствием W3C** (HTML5 валидная разметка, CSS3 валидные стили, все [APG паттерны](https://www.w3.org/WAI/ARIA/apg/patterns/) покрыты).

Справочная react-версия: [`.react-verse/`](.react-verse/) — используется как визуальный ориентир, **не** как архитектурный эталон.

---

## Принятые решения

| Тема | Решение |
|------|---------|
| `data-class` | Не используем. В react показал неэффективность на разных уровнях. |
| `UtilityPropBag` map (react) | Не переносим. Динамическая карта невозможно покрыть адекватно без TS-типизации и линтера. Go-подход: расширяемый `type UtilityProps struct` в [`utils/props.go`](utils/props.go). |
| JavaScript | Используем. Один общий `ui8kit.js` с модульной структурой, подключаемой по мере появления компонентов. Покрывает фокус-менеджмент, клавиатурную навигацию, live regions — то, что невозможно CSS-only. |
| CSS-only хаки (`checkbox+peer`) | Отменяются. Раз JS есть — делаем правильно через JS + ARIA, а не эмулируем чекбоксами. |
| Пропсы для Image | Реализуем внутри компонента (`fit`, `position`, `aspect`), расширяем variant-функции в Go. |
| Валидация | HTML5 (`nu validator`), CSS3 (`W3C CSS Validator`), ARIA APG (ручной чеклист + axe-core). |

## 1. Паритет компонентов с react-версией

### 1.1 Новые компоненты (нет в `ui/`)

- [x] **Image** — `src`, `alt`, `width`, `height`, `fit` (contain/cover/fill/none/scale-down), `position` (center/top/bottom/left/right и комбинации), `aspect` (auto/square/video); variant-функции `ImageFitVariant()`, `ImagePositionVariant()`, `ImageAspectVariant()` в `utils/variants.go`
- [x] **Grid** — `display: grid`, `Cols` (число 1–12), `Gap` (через UtilityProps); responsive-пресеты как variant-функция `GridColsVariant()`
- [x] **GridCol** — `Span`, `Start`, `End`, `Order`; variant-функция `GridColVariant()`
- [x] **Card** — обёртка, `Variant` (совместим с `CardVariant()` из `variants.go`)
- [x] **CardHeader** — `<div>` секция
- [x] **CardTitle** — `Order` 1–6 → тег `<h1>`–`<h6>`
- [x] **CardDescription** — `<p>`
- [x] **CardContent** — `<div>` секция
- [x] **CardFooter** — `<div>` секция
- [ ] **Accordion** — контейнер; `Type` single/multiple; разметка: `<h3>` + `<button aria-expanded aria-controls>` + `<div role="region" aria-labelledby>`; поведение раскрытия через JS модуль
- [x] **AccordionItem** — `Value` (уникальный id секции)
- [x] **AccordionTrigger** — кнопка-заголовок; ARIA: `aria-expanded`, `aria-controls="panel-{value}"`
- [x] **AccordionContent** — панель; ARIA: `id="panel-{value}"`, `role="region"`, `aria-labelledby="trigger-{value}"`
- [ ] **Sheet** — боковая панель; `Side` (left/right), `Size` (sm/md/lg/xl/2xl/full); разметка: `<dialog>` или `role="dialog" aria-modal="true" aria-labelledby`; открытие/закрытие и фокус-трап — JS модуль

### 1.2 Доработка существующих компонентов

- [x] **Box** — рендерить `Tag` (сейчас всегда `<div>`, поле `Tag` в `BoxProps` игнорируется)
- [x] **Block** — alias Box, унаследует исправленный Tag
- [x] **Stack** — рендерить `Tag`; дефолтные классы вынести в helper для удобства переопределения через `UtilityProps`
- [x] **Group** — рендерить `Tag`
- [x] **Container** — оставить `<div>`, добавить `Tag` если потребуется
- [x] **Button** — дефолт `type="button"` (уже есть в helpers); при `Disabled` + `Href`: `aria-disabled="true"`, `tabindex="-1"`, `role="link"`
- [ ] **Field** — добавить `Label`, `Hint`, `Error` поля в `FieldProps` для генерации обёртки `<label>` + `<small>` / `<span role="alert">`; добавить `component="button"` вариант; для checkbox/radio: поддержка `role="switch"` через отдельный проп
- [ ] **Title** — разрешить `{ children... }` вместо только строкового `value` (для вложенных элементов, напр. `<a>` внутри заголовка)
- [x] **Text** — рендерить `Tag` (сейчас всегда `<p>`, иногда нужен `<span>`, `<small>`, `<time>`)
- [x] **Badge** — без изменений (соответствует react)
- [x] **Icon** — без изменений (icon font `latty-*`; SVG-режим не в скоупе MVP)

### 1.3 `utils/` расширения

- [x] Новые variant-функции: `ImageFitVariant`, `ImagePositionVariant`, `ImageAspectVariant`, `GridColsVariant`, `GridColVariant`
- [x] Расширить `UtilityProps` (при необходимости): `Object` (object-fit/position), `Aspect`, `MinH`, `MaxH`
- [x] Вынести ARIA-хелперы (например `AriaExpanded(bool) templ.Attributes`, `AriaControls(id string) templ.Attributes`) в `utils/aria.go` для переиспользования

## 3. JS-архитектура: `ui8kit.js`

Единый entry-point `ui8kit.js` с модульной структурой. Каждый модуль — самоинициализирующийся через `data-ui8kit="<pattern>"` атрибуты на корневом элементе компонента.

```
ui8kit.js
├── core.js          — регистрация модулей, MutationObserver для hot-init
├── accordion.js     — toggle, клавиатура
├── dialog.js        — open/close, focus-trap, Escape, restore focus (Sheet, AlertDialog)
├── tabs.js          — activation, стрелки
├── menu.js          — фокус, подменю, Escape (Menu, MenuBar, MenuButton)
├── combobox.js      — фильтр, popup, activedescendant
├── listbox.js       — выбор, typeahead
├── radiogroup.js    — roving tabindex
├── slider.js        — drag, клавиатура (Slider, RangeSlider)
├── spinbutton.js    — increment/decrement
├── tooltip.js       — hover/focus delay, Escape
├── carousel.js      — slides, autoplay, aria-live
├── tree.js          — expand/collapse, навигация
├── treegrid.js      — tree + grid навигация
├── grid.js          — 2D cell навигация (interactive DataGrid)
├── toolbar.js       — roving tabindex
├── splitter.js      — drag resize, клавиатура
├── feed.js          — lazy load, aria-busy
└── alert.js         — dynamic insertion, live region
```

Подключение: в Templ-шаблон включается `<script src="ui8kit.js">` один раз; JS модули активируются только для компонентов, присутствующих в DOM (через `data-ui8kit` + `MutationObserver`).

## 4. Порядок реализации

### Фаза 1 — Ядро (паритет с react + базовые APG)

1. Исправить полиморфизм `Tag` в Box/Stack/Group/Text (Landmarks закрываются автоматически)
2. Image + variant-функции
3. Grid + GridCol + variant-функции
4. Card + CardHeader/Title/Description/Content/Footer
5. Button: `aria-disabled` + `tabindex` для disabled link
6. Field: label/hint/error обёртка, `role="switch"` вариант
7. `utils/aria.go` — ARIA-хелперы для `templ.Attributes`

### Фаза 2 — Интерактивные паттерны (JS)

1. `core.js` — модульный загрузчик
2. Accordion + `accordion.js`
3. Sheet (Dialog) + `dialog.js`
4. Tabs + `tabs.js`
5. Alert / AlertDialog (переиспользует `dialog.js`)
6. Tooltip + `tooltip.js`

### Фаза 3 — Формы и навигация

1. Breadcrumb
2. RadioGroup + `radiogroup.js`
3. Listbox + `listbox.js`

### Фаза 4 — Продвинутые виджеты

1. Slider / RangeSlider + `slider.js`
2. Spinbutton + `spinbutton.js`

4. Carousel + `carousel.js`

## 5. Критерии приёмки для каждого компонента

- [ ] Валидный HTML5 (проверка `nu validator`)
- [ ] Валидный CSS3 (проверка `W3C CSS Validator`)
- [ ] Соответствие APG для соответствующего паттерна (роли, состояния, свойства, клавиатура)
- [ ] Тест `axe-core` без нарушений на демо-странице компонента
- [ ] Go unit-тест рендера (проверка ключевых атрибутов в HTML-выводе)
- [ ] Документация: пропсы, примеры использования, ARIA-спецификация
